01 — Contexte

Présentation du projet

Dans le cadre de mon stage de BTS SIO SLAM, j'ai participé au développement d'un site web pour le Judo Club Almont. Ma mission principale consistait à concevoir et intégrer un formulaire d'inscription en ligne permettant aux futurs membres de rejoindre le club.

Le projet a été développé avec le framework Laravel, en suivant l'architecture MVC (Modèle – Vue – Contrôleur). L'environnement de développement local utilisé était Laragon, et le versioning a été géré via GitHub avec une convention de branches par fonctionnalité.

Laravel (PHP) Blade (templates) HTML5 / CSS3 Tailwind CSS MySQL Laragon GitHub
02 — Environnement de développement

Mise en place du projet

Avant tout développement, l'environnement local a été configuré selon les consignes fournies. Laragon a été utilisé comme solution tout-en-un (Apache, MySQL, PHP) pour faire tourner Laravel en local.

04 — Base de données

Migration & structure de la table

La table membres a été créée via une migration Laravel, permettant de versionner la structure de la base de données et de la reproduire facilement sur n'importe quel environnement.

Champ Type Description
id bigIncrements Clé primaire auto-incrémentée
nom string Nom de famille du membre
prenom string Prénom du membre
email string, unique Adresse e-mail (identifiant unique)
telephone string, nullable Numéro de téléphone
date_naissance date Date de naissance du membre
categorie string Catégorie d'âge (poussin, benjamin, junior…)
timestamps datetime created_at / updated_at (auto)
database/migrations/create_membres_table.php
Schema::create('membres', function (Blueprint $table) { $table->id(); $table->string('nom'); $table->string('prenom'); $table->string('email')->unique(); $table->string('telephone')->nullable(); $table->date('date_naissance'); $table->string('categorie'); $table->timestamps(); });
05 — Formulaire & validation

Développement du formulaire

Le formulaire d'inscription a été intégré dans une vue Blade et connecté à un contrôleur Laravel gérant la validation des données et l'insertion en base.

01
Affichage du formulaire
La route GET /inscription appelle InscriptionController@create() qui retourne la vue Blade avec le formulaire HTML.
02
Soumission & validation
La route POST /inscription déclenche store(). Laravel valide les champs (required, email, unique, date...) et renvoie les erreurs à la vue si nécessaire.
03
Insertion en base
Si la validation passe, Membre::create($validated) insère le nouveau membre dans la table membres via Eloquent ORM.
04
Redirection & confirmation
L'utilisateur est redirigé avec un message de succès affiché via la session flash Laravel (session('success')).
app/Http/Controllers/InscriptionController.php
public function store(Request $request) { $validated = $request->validate([ 'nom' => 'required|string|max:255', 'prenom' => 'required|string|max:255', 'email' => 'required|email|unique:membres', 'telephone' => 'nullable|string', 'date_naissance' => 'required|date', 'categorie' => 'required|string', ]); Membre::create($validated); return redirect()->route('inscription') ->with('success', 'Inscription envoyée !'); }
06 — Versioning

Workflow Git & GitHub

Le projet a été versionné sur un dépôt GitHub privé. Chaque fonctionnalité a été développée sur une branche dédiée, conformément aux consignes du projet.

# Création d'une branche pour la fonctionnalité git checkout -b feature/formulaire-inscription # Commit du travail git add . git commit -m "feat: ajout formulaire inscription membre" # Push et création de la Pull Request git push origin feature/formulaire-inscription

Une Pull Request a ensuite été soumise vers la branche main pour validation par le responsable du projet.
